To transform the Donors page from a simple list into a "Product-Grade" Emergency Hub, you should focus on three pillars: Compatibility Intelligence,
Geographic Proximity, and Frictionless Communication.

Here are detailed suggestions for the revamp:

1. The Interactive Compatibility Explorer
   Instead of just showing counts, make the page a "Matchmaker."

- Two-Way Logic: When a user selects a blood group (e.g., B+), the interface should instantly highlight which other groups can Receive from them and
  which they can Give to. This educates the user during a stressful emergency.
- Smart Highlighting: Use a "Focus Mode" where selecting one blood tile dims the others, except for the compatible ones. This reduces visual noise.

2. Emergency "Mass Action" Tools
   In a real medical crisis, every second counts.

- The "Copy All" Payload: Add a button to "Copy All Mobile Numbers" for a specific blood group. This allows a student to paste 10+ numbers into their SMS
  app or a WhatsApp group in one go.
- Priority Ranking: Order the donor list within each group. For example, if the user is in Rajshahi, show donors whose hometown is "Rajshahi" at the top.

3. Geographic Awareness (Proximity Filter)
   Blood is needed where the patient is.

- Location Badges: Next to every donor's name in the list, show their Hometown.
- "In-City" Filter: Add a quick toggle to "Show only donors currently in Rajshahi" (assuming most students are at the university). This prevents calling
  someone who is currently across the country during a break.

4. Visual Hierarchy & "Rare" Alerts

- Scarcity Badges: Automatically tag groups with 1–2 donors (like AB- or O-) as "Rare Signal". This alerts the batch that these specific individuals are
  critical resources.
- Visual Pulse: Use a "Revolut-style" progress bar on each blood tile to show how many donors are available relative to the largest group (usually O+ or
  A+). This gives a quick "Batch Health" overview.

5. Frictionless UI (The "Drawer" Pattern)

- Side Explorer: Instead of jumping back to the main roster, clicking a blood group should open a Side Drawer or Expandable Panel right there.
- Quick Info: Inside that panel, list the names, rolls, and phone numbers with "Tap-to-Call" buttons. The goal is to never make the user leave the Donors
  page to get a number.

6. "Signal Readiness" Status

- Data Completeness: Show a "Batch Pulse" meter—e.g., "94% of Batch 24 has verified blood groups." This encourages the remaining 6% to provide their data
  to the CR.
- The "Last Updated" Ghost: A subtle timestamp at the bottom of the donor list to build trust that the data is current.

Summary of the UX Goal:
The user should be able to: Enter → Select recipient type → See compatible donors → Filter by location → Copy all numbers—all in under 10 seconds.
